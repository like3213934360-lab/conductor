# LSO Quickstart — 智能路由入口

## 触发条件
- 用户不确定从哪个工作流开始
- 首次使用 Liquid Swarm Orchestrator
- 项目状态不明

## 执行步骤

### Step 1: 项目状态诊断
检测以下指标：
1. `.anws/` 目录是否存在 → 判断是否已初始化架构
2. `.anws/v{N}/05_TASKS.md` 是否存在 → 判断是否已有任务清单
3. `src/` 目录是否有代码 → 判断是否已有实现
4. Git 历史是否存在 → 判断是新项目还是已有项目

### Step 2: 智能路由
根据诊断结果，推荐以下工作流：

| 状态 | 推荐工作流 |
|------|-----------|
| 无 `.anws/`，无代码 | → `/LSO-genesis`（从零启动）|
| 有代码，无 `.anws/` | → `/LSO-probe`（遗留代码分析）|
| 有 `.anws/`，无任务 | → `/LSO-blueprint`（任务拆解）|
| 有任务清单 | → `/LSO-forge`（开始编码）|
| 需要多模型分析 | → `/LSO-swarm-analyze`（三态蜂群分析）|

### Step 3: 确认并跳转
向用户展示诊断结果和推荐，等待确认后跳转到对应工作流。
